# Issue Conventions

## Bug Reports

- Title: clear symptom description
- Body: steps to reproduce, expected vs actual behavior, environment details
- Error messages verbatim in code blocks

## Feature Requests

- Title: desired outcome, not implementation detail
- Body: motivation/use case, proposed approach (if any), alternatives considered

## General

- Use GitHub-flavored markdown (task lists, code blocks with language hints)
- Use permalink URLs when referencing specific lines of code
- Don't add labels unless user specifies or repo has documented label conventions
- Don't assume label names — list available labels first if labeling is needed
