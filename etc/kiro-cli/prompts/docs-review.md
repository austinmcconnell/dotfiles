# Docs Review

Review this documentation repository using the docs-review skill. Follow the full review workflow
(Steps 1-7) and generate a review report covering:

1. Content ownership violations (WHAT/HOW/WHY/SPECS separation)
1. Duplicate specifications across files
1. Cross-reference completeness and link quality
1. SUMMARY.md accuracy against actual files
1. README.md completeness for each subdirectory

Flag specific files and line numbers for each issue found. Prioritize findings as high/medium/low in
the final report.

Write the full report as a markdown file to `reports/docs-review-YYYY-MM-DD.md` (using today's
date). Create the `reports/` directory if it doesn't exist. Provide a brief summary in chat.
